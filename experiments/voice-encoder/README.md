# Kokoro Voice Encoder Experiment

This is an isolated research scaffold for learning a native Kokoro voice style
predictor:

```text
reference audio -> log-mel/speaker encoder -> latent -> Kokoro style table
```

The goal is to generate a custom Kokoro-compatible voicepack and avoid running
Kanade voice conversion in the FastKokoro hot path.

This directory is intentionally outside the package runtime. Do not add torch,
torchaudio, or training dependencies to the base `fastkokoro` package until the
approach is proven.

## Milestones

1. Collect public datasets and audit licenses.
2. Build a synthetic Kokoro dataset from official voicepacks.
3. Train a first encoder that recovers official voicepacks from generated audio.
4. Add triple loss against style, speaker similarity, and mel/audio perception.
5. Test zero-shot behavior with public real speakers.
6. Export `voices-custom.npz` and load it with `FASTKOKORO_VOICES_PATH`.

## Data Sources

Public sources mentioned by Kokoro materials:

- SIWIS: https://datashare.ed.ac.uk/handle/10283/2353
- Koniwa: https://github.com/koniwa/koniwa
- Voicepack dataset: https://huggingface.co/datasets/ecyht2/kokoro-82M-voices

The full Kokoro training dataset does not appear to be published. Treat the
public sources as evaluation/fine-tuning material, not a full reconstruction of
the original training set.

## Container

Build a research image:

```bash
docker build -f experiments/voice-encoder/Dockerfile -t fastkokoro-voice-encoder .
```

Run with GPU:

```bash
docker run --rm -it --gpus all \
  -v "$PWD":/workspace \
  -w /workspace \
  fastkokoro-voice-encoder bash
```

Run without GPU:

```bash
docker run --rm -it \
  -v "$PWD":/workspace \
  -w /workspace \
  fastkokoro-voice-encoder bash
```

## Synthetic Dataset

Create a synthetic dataset from FastKokoro itself:

```bash
python experiments/voice-encoder/scripts/generate_synthetic_dataset.py \
  --config experiments/voice-encoder/config.example.json
```

Outputs:

- `experiments/voice-encoder/data/synthetic/audio/*.wav`
- `experiments/voice-encoder/data/synthetic/manifest.jsonl`

Each manifest row includes:

- voice id
- language
- text
- phonemes
- token count
- selected Kokoro style index
- generated audio path
- target style shape

## Voicepack Inspection

Inspect an NPZ voicepack:

```bash
python experiments/voice-encoder/scripts/inspect_voicepacks.py \
  --voices-path /path/to/voices-fastkokoro.npz
```

## First Training Target

Start with a constrained task:

```text
audio generated by voice X -> predict style[token_count] for voice X
```

Then expand to:

```text
audio generated by voice X -> predict full style table for voice X
```

Recommended initial losses:

```text
loss = a * L_style + b * L_speaker + c * L_recon
```

- `L_style`: MSE/cosine against Kokoro style target.
- `L_speaker`: speaker embedding similarity between reference and generated
  audio.
- `L_recon`: log-mel or perceptual reconstruction loss after Kokoro synthesis.

Keep Kanade as an offline teacher candidate only after the official voicepack
recovery task converges.
